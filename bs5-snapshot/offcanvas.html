Offcanvas

An edge-attached sheet for side panels, drawers, and quick captures.

Basic

A trigger opens the offcanvas via data-bs-toggle="offcanvas" and data-bs-target. Inside the panel: an optional .offcanvas-header with the title, an .offcanvas-body for content, and an optional .offcanvas-footer for sticky actions. The dismiss control is .offcanvas-close, a ghost icon button that sits at the trailing edge of the header row.

Placements

Four placement classes anchor the sheet to a screen edge. .offcanvas-start slides in from the left, .offcanvas-end from the right, .offcanvas-top from above, and .offcanvas-bottom from below. Width is fixed for start and end, height is fixed for top and bottom.

Static backdrop

Set data-bs-backdrop="static" and data-bs-keyboard="false" to force a deliberate dismiss. Use this when the sheet must be completed before the user moves on, like a setup flow or a required confirmation.

No backdrop

Responsive

Add .offcanvas-{sm|md|lg|xl|xxl} to make the sheet collapse only below the named breakpoint. Above it the markup renders inline in place, so the same template carries a drawer on mobile and an inline panel on desktop. Pair the trigger with .d-{breakpoint}-none so it hides once the layout goes inline.

Resize the window to see the switch. The demo uses .offcanvas-lg, so below the lg breakpoint the Menu button opens a drawer; at lg and up the section list renders inline.
