Buttons

A clickable control that triggers an action.

Primary

The main call to action. Use one per surface for the action you most want the user to take, like Save, Submit, or Continue. 

<button type="button" class="btn btn-primary">Primary</button>

Secondary

For actions that sit alongside primary but should not compete with it. Common cases are Cancel next to Save, Filter next to Search, or any non-prominent control.

Two flavors. The filled secondary reads as a peer to primary, useful when both actions are equal weight. The outline secondary is quieter and sits in the grey ramp, which is the canonical Cancel-next-to-Save pairing. 

<div class="d-flex flex-wrap gap-2">
  <button type="button" class="btn btn-secondary">Secondary</button>
  <button type="button" class="btn btn-outline-secondary">Outline</button>
</div>

The canonical pairing in context.

<div class="d-flex flex-wrap gap-2">
  <button type="button" class="btn btn-primary">Save changes</button>
  <button type="button" class="btn btn-outline-secondary">Cancel</button>
</div>

Danger

For destructive actions like Delete, Remove, or Discard. The filled red carries the most weight and is the safe default for a confirm dialog. Outline and light are options when a softer treatment fits the surface better, like a row-level delete inside a dense table. 

<div class="d-flex flex-wrap gap-2">
  <button type="button" class="btn btn-danger">Danger</button>
  <button type="button" class="btn btn-outline-danger">Outline</button>
  <button type="button" class="btn btn-light-danger">Light</button>
</div>

Tertiary

An alternative to primary for surfaces where the action should be prominent but the brand primary color doesn't fit. Tertiary's color flips by theme. Dark in light mode, light in dark mode. Reach for it when primary is already taken on the page and a second prominent action needs to stand apart. 

<button type="button" class="btn btn-tertiary">Tertiary</button>

Ghost

No background, no border, just a colored label. Use it for low-stakes actions like a modal Close, a Cancel inside a card row, or toolbar items. Hover paints a soft tint so the affordance shows up on interaction. 

<button type="button" class="btn btn-ghost">Ghost</button>

Other variants

Stisla v3 ships filled, outline, light, and ghost flavors for every Bootstrap theme color. You probably won't need most of these in a real app. Reach for primary, secondary, danger, tertiary, or ghost first. Everything below is here for the rare case a surface calls for it.

Where are btn-light and btn-dark?

Stisla v3 doesn't restyle BS5's .btn-light or .btn-dark. In practice they don't pull their weight as semantic categories. .btn-light reads as a barely-visible grey button on a white surface, and .btn-dark is essentially .btn-primary inverted. BS5's compiled defaults still emit, so existing markup keeps working, just unbranded.

Use btn-secondary instead. It's mono and inverts per theme, which is what .btn-dark (light mode) and .btn-light (dark mode) were trying to be. Heads-up on the name collision. The light flavors in Other variants above (btn-light-primary, btn-light-success, ...) are unrelated. Same word, different family. They're per-color soft tints, not the BS5 grey button.

Link button

No background, no shadow. Underline on hover.

<button type="button" class="btn btn-link">Link button</button>

Sizes

Heights are pinned across small, default, and large regardless of content.

<div class="d-flex flex-wrap gap-2 align-items-center">
  <button type="button" class="btn btn-primary btn-sm">Small</button>
  <button type="button" class="btn btn-primary">Default</button>
  <button type="button" class="btn btn-primary btn-lg">Large</button>
</div>

With icon

Any icon component drops into .btn via flexbox gap. Icons scale with the button's font size. 

<div class="d-flex flex-wrap gap-2 align-items-center">
  <button type="button" class="btn btn-primary">
    <i data-lucide="plus"></i>
    Leading icon
  </button>
  <button type="button" class="btn btn-primary">
    Trailing icon
    <i data-lucide="arrow-right"></i>
  </button>
</div>

Icon only

Add .btn-icon for a 1:1 square button. Pair with aria-label so the action stays announced. 

<div class="d-flex flex-wrap gap-2 align-items-center">
  <button type="button" class="btn btn-primary btn-icon btn-sm" aria-label="Add">
    <i data-lucide="plus"></i>
  </button>
  <button type="button" class="btn btn-primary btn-icon" aria-label="Add">
    <i data-lucide="plus"></i>
  </button>
  <button type="button" class="btn btn-primary btn-icon btn-lg" aria-label="Add">
    <i data-lucide="plus"></i>
  </button>
  <button type="button" class="btn btn-outline-danger btn-icon" aria-label="Delete">
    <i data-lucide="trash-2"></i>
  </button>
  <button type="button" class="btn btn-outline-secondary btn-icon" aria-label="Edit">
    <i data-lucide="pencil"></i>
  </button>
</div>

States

Disabled link 
<div class="d-flex flex-wrap gap-2 align-items-center">
  <button type="button" class="btn btn-primary active" aria-pressed="true">Active</button>
  <button type="button" class="btn btn-primary" disabled>Disabled button</button>
  <a href="#" role="button" class="btn btn-primary disabled" aria-disabled="true" tabindex="-1">Disabled link</a>
</div>

Loading

Toggle .btn-loading from JS to show a leading spinner. The label stays visible. Any leading or trailing icons are hidden so the spinner takes their place. Click is blocked while the class is applied. Pair with aria-busy="true". 

<div class="d-flex flex-wrap gap-2 align-items-center">
  <button type="button" class="btn btn-primary btn-loading btn-sm" aria-busy="true">Saving</button>
  <button type="button" class="btn btn-primary btn-loading" aria-busy="true">Saving</button>
  <button type="button" class="btn btn-primary btn-loading btn-lg" aria-busy="true">Saving</button>
  <button type="button" class="btn btn-outline-secondary btn-loading" aria-busy="true">Loading</button>
  <button type="button" class="btn btn-danger btn-icon btn-loading" aria-busy="true" aria-label="Deleting"></button>
</div>

With icons, the spinner replaces them in place.

<div class="d-flex flex-wrap gap-2 align-items-center">
  <button type="button" class="btn btn-primary btn-loading" aria-busy="true">
    <i data-lucide="plus"></i>
    Adding
  </button>
  <button type="button" class="btn btn-primary btn-loading" aria-busy="true">
    Continuing
    <i data-lucide="arrow-right"></i>
  </button>
</div>

Click to try. The state auto-clears after two seconds.

<button type="button" class="btn btn-primary" data-demo-loading="2000">
  <i data-lucide="save"></i>
  Save changes
</button>

Toggle

data-bs-toggle="button" flips .active and aria-pressed on click. 

<button type="button" class="btn btn-primary" data-bs-toggle="button">Press me</button>

Works on any element

.btn renders the same on <button>, <a>, and form inputs.

<a> 
<div class="d-flex flex-wrap gap-2 align-items-center">
  <button type="button" class="btn btn-primary">&lt;button&gt;</button>
  <a href="#" role="button" class="btn btn-primary">&lt;a&gt;</a>
  <input type="submit" class="btn btn-primary" value="<input type=submit>" />
</div>
