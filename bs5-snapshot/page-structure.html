Page structure

The structural component for what goes inside <main>.

The page wrapper

.page is a flex-column that owns the gap between its children via --bs-page-section-gap (default 1.5rem). The header and each section carry no outer margin themselves.

Customers

First section content.

Second section content.

<div class="page p-4 border rounded bg-body w-100">
  <header class="page-header">
    <div class="page-header-title">
      <h1 class="h3 mb-0">Customers</h1>
    </div>
  </header>
  <section class="page-section">
    <p class="text-body-secondary mb-0">First section content.</p>
  </section>
  <section class="page-section">
    <p class="text-body-secondary mb-0">Second section content.</p>
  </section>
</div>

Page header

.page-header is a flex row with a .page-header-title slot on the leading edge and a .page-header-action slot on the trailing edge. Both slots are passthrough. Title takes an h1 with an optional subtitle, actions takes any buttons.

Customers

All accounts in this workspace.

<div class="page p-4 border rounded bg-body w-100">
  <header class="page-header">
    <div class="page-header-title">
      <h1 class="h3 mb-1">Customers</h1>
      <p class="text-body-secondary mb-0">All accounts in this workspace.</p>
    </div>
    <div class="page-header-action">
      <button type="button" class="btn btn-outline-secondary"><i data-lucide="download" class="me-1"></i>Export</button>
      <button type="button" class="btn btn-primary"><i data-lucide="plus" class="me-1"></i>Add customer</button>
    </div>
  </header>
</div>

Wraps on narrow widths

The header is flex-wrap: wrap. When the viewport gets too narrow for the title and actions to share a row, the actions drop below.

<header class="page-header">
  <div class="page-header-title">
    <h1 class="h3 mb-1">Customers</h1>
    <p class="text-body-secondary mb-0">All accounts in this workspace.</p>
  </div>
  <div class="page-header-action">
    <button type="button" class="btn btn-primary">Add customer</button>
  </div>
</header>

Sections

.page-section is a flex-column for each block inside the page. The .page parent supplies the gap between sections, and each section supplies its own inner gap via --bs-page-section-inner-gap.

With section header + actions

For sections that need an action row of their own, wrap the title in .page-section-header and reuse .page-header-action as the slot. Same recipe at a tighter scale.

Breadcrumb placement

Put the breadcrumb as the first child of .page, above .page-header. It inherits the page's gap automatically.

  1. Dashboard
  2. Reports
  3. Monthly

Monthly

<div class="page p-4 border rounded bg-body w-100">
  <nav aria-label="breadcrumb">
    <ol class="breadcrumb mb-0">
      <li class="breadcrumb-item"><a href="#">Dashboard</a></li>
      <li class="breadcrumb-item"><a href="#">Reports</a></li>
      <li class="breadcrumb-item active" aria-current="page">Monthly</li>
    </ol>
  </nav>
  <header class="page-header">
    <div class="page-header-title">
      <h1 class="h3 mb-0">Monthly</h1>
    </div>
  </header>
</div>

Skip the breadcrumb on top-level pages where it would only show one crumb.

Content width

.page is unopinionated about width. Wrap it (or set its own classes) to clamp the content column.

  • Full width (.container-fluid or no container) for dashboards, tables, anything dense.
  • Breakpoint-clamped (.container) for marketing pages, or anything that should match Bootstrap's standard column widths.
  • Custom max-width (40 to 60rem) for settings forms, articles, anything text-heavy where long lines hurt readability.
<!-- Full width -->
<main class="app-shell-main">
  <div class="page container-fluid p-4">...</div>
</main>

<!-- Breakpoint-clamped -->
<main class="app-shell-main">
  <div class="page container py-4">...</div>
</main>

<!-- Custom narrow column for reading -->
<main class="app-shell-main">
  <div class="page mx-auto p-4" style="max-width: 48rem;">...</div>
</main>
