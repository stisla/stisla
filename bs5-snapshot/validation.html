Validation

Status styles and feedback messages for form fields, driven by either the browser or your own server response.

States

Add .is-valid or .is-invalid to any form control to mark the field. Pair with a sibling .valid-feedback or .invalid-feedback for the message. Use this pattern when state comes back from a server response or your own client logic.

Looks good.
Enter a complete email address.
<div class="d-flex flex-column gap-3 w-100" style="max-width: 24rem;">
  <div>
    <label for="validName" class="form-label">Name</label>
    <input type="text" class="form-control is-valid" id="validName" value="Ada Lovelace" />
    <div class="valid-feedback">Looks good.</div>
  </div>
  <div>
    <label for="invalidEmail" class="form-label">Email</label>
    <input type="email" class="form-control is-invalid" id="invalidEmail" value="ada@" />
    <div class="invalid-feedback">Enter a complete email address.</div>
  </div>
</div>

Browser validation

Add .was-validated to the <form> after a submit attempt to light up native HTML5 constraints (required, type="email", minlength, etc.) without per-field classes. Add novalidate to suppress the browser's own bubbles and let the inline feedback do the work.

Name is required.
Enter a valid email.
<form class="was-validated d-flex flex-column gap-3 w-100" style="max-width: 24rem;" novalidate>
  <div>
    <label for="nativeName" class="form-label">Full name</label>
    <input type="text" class="form-control" id="nativeName" value="Grace Hopper" required />
    <div class="invalid-feedback">Name is required.</div>
  </div>
  <div>
    <label for="nativeEmail" class="form-label">Email</label>
    <input type="email" class="form-control" id="nativeEmail" required />
    <div class="invalid-feedback">Enter a valid email.</div>
  </div>
</form>

On submit

Listen for submit, call checkValidity(), and add .was-validated to flip the form into its validated state. preventDefault() keeps the browser from posting so you can hand off to fetch once the form is clean.

Name is required.
Enter a valid email.
You must agree before continuing.
<form id="submitForm" class="d-flex flex-column gap-3 w-100" style="max-width: 24rem;" novalidate>
  <div>
    <label for="submitName" class="form-label">Full name</label>
    <input type="text" class="form-control" id="submitName" required />
    <div class="invalid-feedback">Name is required.</div>
  </div>
  <div>
    <label for="submitEmail" class="form-label">Email</label>
    <input type="email" class="form-control" id="submitEmail" required />
    <div class="invalid-feedback">Enter a valid email.</div>
  </div>
  <div class="form-check">
    <input class="form-check-input" type="checkbox" id="submitTerms" required />
    <label class="form-check-label" for="submitTerms">I agree to the terms</label>
    <div class="invalid-feedback">You must agree before continuing.</div>
  </div>
  <button type="submit" class="btn btn-primary align-self-start">Sign up</button>
</form>
<script>
  (function () {
    const form = document.getElementById('submitForm');
    form.addEventListener('submit', (event) => {
      event.preventDefault();
      form.classList.add('was-validated');
      if (form.checkValidity()) {
        // hand off to fetch / your API
      }
    });
  })();
</script>

Tooltip feedback

Swap .valid-feedback / .invalid-feedback for .valid-tooltip / .invalid-tooltip to float the message over the layout instead of pushing siblings down. The parent needs position: relative.

Available.
City is required.
<div class="d-flex flex-column gap-4 w-100" style="max-width: 24rem;">
  <div class="position-relative">
    <label for="tipUser" class="form-label">Username</label>
    <input type="text" class="form-control is-valid" id="tipUser" value="ada" />
    <div class="valid-tooltip">Available.</div>
  </div>
  <div class="position-relative">
    <label for="tipCity" class="form-label">City</label>
    <input type="text" class="form-control is-invalid" id="tipCity" />
    <div class="invalid-tooltip">City is required.</div>
  </div>
</div>

Select

The same classes work on .form-select.

Region set.
Pick a plan to continue.
<div class="d-flex flex-column gap-3 w-100" style="max-width: 24rem;">
  <div>
    <label for="validRegion" class="form-label">Region</label>
    <select class="form-select is-valid" id="validRegion">
      <option>Asia Pacific</option>
    </select>
    <div class="valid-feedback">Region set.</div>
  </div>
  <div>
    <label for="invalidPlan" class="form-label">Plan</label>
    <select class="form-select is-invalid" id="invalidPlan">
      <option value="" selected>Choose a plan</option>
      <option value="starter">Starter</option>
      <option value="pro">Pro</option>
    </select>
    <div class="invalid-feedback">Pick a plan to continue.</div>
  </div>
</div>

Checks & radios

Apply the state class to the .form-check-input. Feedback can sit alongside the label inside the same .form-check.

You must agree before continuing.
Preference saved.
<div class="d-flex flex-column gap-3 w-100" style="max-width: 24rem;">
  <div class="form-check">
    <input class="form-check-input is-invalid" type="checkbox" id="agreeTerms" required />
    <label class="form-check-label" for="agreeTerms">I agree to the terms</label>
    <div class="invalid-feedback">You must agree before continuing.</div>
  </div>
  <div class="form-check">
    <input class="form-check-input is-valid" type="checkbox" id="agreeMarketing" checked />
    <label class="form-check-label" for="agreeMarketing">Send me product updates</label>
    <div class="valid-feedback">Preference saved.</div>
  </div>
</div>

File input
