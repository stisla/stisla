Sidebar

A vertical navigation panel with no fixed position, no width, and no background by default. Your layout decides where it sits and how wide it gets.

Basic

A .sidebar wraps .sidebar-header, .sidebar-content, and an optional .sidebar-footer. Items live inside .sidebar-menu, which holds one or more .sidebar-group, each containing a .sidebar-list.

Item actions

Place a .sidebar-item-action after the button to drop a badge or a small button into the right edge of the row. The main button keeps its full click area; the action sits on top of the right edge and only catches clicks on itself. Add .sidebar-item-action-hover to only show the action on row hover or keyboard focus. This works well for dense lists where always-visible actions feel noisy.

Nested sub-menu

Rail / mini mode

Add .is-collapsed to the .sidebar to shrink it down to icons only. Each .sidebar-item-button becomes a square with its icon centered. Labels, submenus, and chevrons fade out. Wrap label text in a <span> so it can be hidden with CSS. Bare text nodes cannot be targeted directly; the button's overflow: hidden still clips them as a fallback, but a span is cleaner.

The button size follows the sidebar size modifiers like .sidebar-sm and .sidebar-lg. The panel's width is still your layout's job. Match it to the item size for a clean fit. Stisla v3 does not include a sidebar toggle script, so you wire that up yourself. See the next section for a working example.

<script>
  (function () {
    const sidebar = document.getElementById('demo-toggleable-sidebar');
    const trigger = document.getElementById('demo-toggleable-sidebar-trigger');
    trigger.addEventListener('click', () => {
      const willCollapse = !sidebar.classList.contains('is-collapsed');
      if (willCollapse) {
        // Close any open submenus first so they animate shut alongside
        // the width slide instead of snapping to display:none mid-collapse.
        sidebar.querySelectorAll('.collapse.show').forEach((el) => {
          Stisla.bs.Collapse.getOrCreateInstance(el).hide();
        });
      }
      sidebar.classList.toggle('is-collapsed');
      sidebar.style.width = willCollapse ? '4.25rem' : '18rem';
      trigger.setAttribute('aria-expanded', willCollapse ? 'false' : 'true');
    });
  })();
</script>

That is the whole script. From here you can persist the state to localStorage, swap the icon, animate the icon flip, move the trigger to a topbar button, reopen submenus when expanding, and so on. The script also closes any open submenus before collapsing the sidebar. It uses Bootstrap 5's Collapse API (Stisla.bs.Collapse when you load Stisla's bundle, bootstrap.Collapse if you import Bootstrap separately) so the submenu height animation runs alongside the width slide. Without that step, the display: none on collapsed submenus would snap them shut instead of animating.

Dark mode

Item hover and active surfaces tint --bs-emphasis-color-rgb. Bootstrap 5 flips that value in dark mode automatically, so the sidebar follows the theme without a separate dark stylesheet.
