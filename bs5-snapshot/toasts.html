Toasts

A non-blocking notification panel.

Basic

A toast is a small surface with a title and a body message. Keep .show for static demos, the framework JS handles it once you trigger a real toast.

Workspace saved
Your changes to billing details have been written and synced.
<div class="toast show" role="alert" aria-live="polite" aria-atomic="true">
  <div class="toast-header">
    Workspace saved
  </div>
  <button type="button" class="btn-close" data-bs-dismiss="toast" aria-label="Close"></button>
  <div class="toast-body">Your changes to billing details have been written and synced.</div>
</div>

Anatomy

Padding lives on .toast itself, so the header band and body padding from Bootstrap drop to zero. The layout is a 2-column grid. Col 1 stacks .toast-header, .toast-body, and .toast-action. Col 2 holds .btn-close as a direct child of .toast so it sits in its own cell at the top-right corner.

Action row

Add .toast-action below the body for inline links or buttons. The row sits at the root's left padding edge, aligned with the leading slot above. For ghost buttons, add .btn-flush-start so the label text lines up with the body above instead of the button frame.

Trigger

Bootstrap ships the toast as opt-in JS. Mount the markup once inside a .toast-container, then call .show() on the instance to reveal it. The container fixes the toast to a corner of the viewport.

Link copied
Anyone with the link can view the page.
<button type="button" class="btn btn-primary" data-demo-toast="#trigger-toast">Show toast</button>
<div class="toast-container position-fixed top-0 end-0 p-3">
  <div class="toast fade" id="trigger-toast" role="status" aria-live="polite" aria-atomic="true">
    <div class="toast-header">
      <i data-lucide="link" class="text-success"></i>
      Link copied
    </div>
    <button type="button" class="btn-close" data-bs-dismiss="toast" aria-label="Close"></button>
    <div class="toast-body">Anyone with the link can view the page.</div>
  </div>
</div>

The button above uses a site-only data-demo-toast attribute to fire the toast. In your own code, grab the element and call the API directly.

trigger.js
const el = document.querySelector('#trigger-toast');
const toast = Stisla.bs.Toast.getOrCreateInstance(el, {
  autohide: true,
  delay: 4000,
});
toast.show();
