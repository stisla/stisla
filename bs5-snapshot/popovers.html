Popovers

A small floating panel anchored to a trigger.

Basic

Set data-bs-toggle="popover" on any focusable element and pass the title and body via data-bs-title and data-bs-content. The popover initializes from a one-line scanner in site.js.

<button type="button" class="btn btn-primary" data-bs-toggle="popover" data-bs-title="API key" data-bs-content="Used to authenticate requests to the workspace from your scripts and CI.">
  Show popover
</button>

Directions

Use data-bs-placement to anchor the panel on a specific side of the trigger.

<div class="d-flex flex-wrap gap-2">
  <button type="button" class="btn btn-outline-secondary" data-bs-toggle="popover" data-bs-placement="top" data-bs-title="Top" data-bs-content="Anchors the panel above the trigger.">
    Top
  </button>
  <button type="button" class="btn btn-outline-secondary" data-bs-toggle="popover" data-bs-placement="right" data-bs-title="Right" data-bs-content="Anchors the panel to the right of the trigger.">
    Right
  </button>
  <button type="button" class="btn btn-outline-secondary" data-bs-toggle="popover" data-bs-placement="bottom" data-bs-title="Bottom" data-bs-content="Anchors the panel below the trigger.">
    Bottom
  </button>
  <button type="button" class="btn btn-outline-secondary" data-bs-toggle="popover" data-bs-placement="left" data-bs-title="Left" data-bs-content="Anchors the panel to the left of the trigger.">
    Left
  </button>
</div>

Hover trigger

Add data-bs-trigger="hover focus" to open on pointer hover and keyboard focus. Click stays the default elsewhere so accidental dismissals don't fight the user.

<button type="button" class="btn btn-outline-secondary" data-bs-toggle="popover" data-bs-trigger="hover focus" data-bs-title="Read-only" data-bs-content="Members with the viewer role can browse but not edit shared boards.">
  Hover or focus me
</button>

Dismiss on next click

Set data-bs-trigger="focus" so the popover closes the moment the trigger loses focus. Render the trigger as an <a> with tabindex="0" so it stays in the tab order without acting like a button.

Click, then click away 
<a tabindex="0" class="btn btn-outline-secondary" role="button" data-bs-toggle="popover" data-bs-trigger="focus" data-bs-title="Dismissible" data-bs-content="Click elsewhere or tab out to close this popover.">
  Click, then click away
</a>

HTML content

Pass data-bs-html="true" to render the content as HTML instead of escaping it. Useful for short inline emphasis or links.

<button type="button" class="btn btn-outline-secondary" data-bs-toggle="popover" data-bs-html="true" data-bs-title="Release notes" data-bs-content="<strong>v3.0</strong> ships the popover, navbar, and offcanvas components. <a href='#'>See changelog</a>.">
  View update
</button>

Disabled triggers

Disabled buttons don't fire pointer events, so wrap them in a focusable span and put the popover attributes on the wrapper.

<span class="d-inline-block" tabindex="0" data-bs-toggle="popover" data-bs-trigger="hover focus" data-bs-content="Reach out to the workspace owner to enable this action.">
  <button type="button" class="btn btn-primary" disabled style="pointer-events: none;">Disabled action</button>
</span>
