Scrollspy

A behavior that highlights nav links as their sections scroll into view.

Basic

Put data-bs-spy="scroll" on the scrollable container and point data-bs-target at the nav. Wrap each section in a <div> whose id matches a nav link's href. The wrapper is the observed element, so Bootstrap can track which section is in view across the full content block, not just the heading line.

Overview Setup Usage Reference

Overview

Scrollspy reports which section is in view by toggling .active on the matching link. It doesn't draw anything itself.

Under the hood it runs an IntersectionObserver against the elements your nav points at. The links can live anywhere on the page; only their href needs to match.

Setup

Give the scroll container data-bs-spy="scroll" and point data-bs-target at the nav whose links should flip.

Each link's href must reference an id that lives inside the scroll container. Bootstrap reads the hash, not the link text.

Usage

Add sections, give them ids, and add nav links to match. Adding or removing sections at runtime requires calling refresh() on the instance after the DOM changes.

The container needs a height and overflow-auto so it actually scrolls. Without those, every section is in view at once and the behavior is a no-op.

Reference

Tunable options are data-bs-root-margin, data-bs-threshold, and data-bs-smooth-scroll. The first two map directly to IntersectionObserver.

The legacy data-bs-offset attribute still works but is deprecated. Prefer data-bs-root-margin for new code.

List group target

The nav doesn't have to be a .nav. Any container whose direct <a> children carry hash hrefs works, including .list-group.

Identity

Set the workspace name, slug, and primary owner. The slug becomes part of every shareable URL the team hands out.

Renaming the workspace updates the display name everywhere but keeps the slug stable, so existing links keep working.

Billing

Manage the plan, payment method, and invoice history. Plan changes take effect at the next billing cycle.

Annual plans get a discount that applies on renewal, not retroactively to the active period.

Members

Invite teammates by email and assign roles. Owners can promote and demote; admins can invite but not delete the workspace.

Pending invitations expire after seven days. Resending refreshes the link without losing the original role assignment.

Audit log

A rolling 90 days of permission changes, exports, and destructive actions. Older entries can be downloaded as CSV.

Each entry records the actor, the target, the action, and the source IP. The log is read-only even to owners.

Smooth scroll

Add data-bs-smooth-scroll="true" and clicking a nav link scrolls to the matching section instead of jumping. The active state still updates from the scroll position, so smooth scroll plays well with manual scrolling.

Step 1 Step 2 Step 3

Step 1

Connect a source. Stisla supports any plain HTTPS endpoint that returns JSON.

The connection is validated on save, so a bad URL fails fast instead of breaking the next sync.

Step 2

Map fields. Each source field can route to one destination column, with an optional transform applied in transit.

Transforms run on the worker, so they don't slow down the sync write. The library covers casts, regex extracts, and date reformatting.

Step 3

Schedule the sync. Cron syntax and the human shorthand both work. Test runs don't count against the quota.

A failing run pauses the schedule and pings the workspace owner. Resuming requires a manual unblock so a broken integration can't drain the budget.

Nested nav

Nested links get nested active states. Use a .nav with sub-.nav children, and Bootstrap marks the parent active when any child is.

Getting started
Install First run
Configuration
Tokens Themes

Getting started

This guide walks through the install and the first successful run. Skip ahead if you've shipped a Stisla project before.

The setup assumes a recent Node version and a bundler that can resolve ESM. Vite, Webpack 5, and Rollup all work out of the box.

Install

Pull the package from npm. Stisla ships an ESM build and a UMD build, so any bundler picks the right one automatically.

Peer dependencies are listed in the package manifest. The framework expects Bootstrap 5.3 as a peer; pin the version explicitly in production.

First run

Import the stylesheet and the JS entry. The framework auto-initializes declarative behaviors on load.

Open any page and inspect the body for the stisla-ready class. If it's missing, the bundle didn't execute and the data-API behaviors will stay dormant.

Configuration

Most projects ship without editing a single SCSS file. When you do need to tweak, override tokens before importing the stylesheet.

The token surface is intentionally narrow. Anything outside the documented variables is implementation detail and may change between minor versions.

Tokens

Color, spacing, radius, and shadow tokens live in a single map. Override the keys you need; the rest cascade from the defaults.

Token overrides resolve at build time. Runtime theming should go through the CSS variables that the tokens generate.

Themes

Light and dark themes are driven by data-bs-theme on <html>. Custom themes follow the same attribute pattern.

A theme is just a scoped block of CSS variables. Defining one is a matter of authoring the overrides and naming the attribute value.

Tuning activation

Under the hood the behavior runs on IntersectionObserver. Two attributes shape when a section counts as "in view".

data-bs-root-margin takes the same string as the IntersectionObserver option. A negative bottom margin like 0px 0px -40% delays activation until the section is well into the visible area, which feels more accurate when sections are short.

data-bs-threshold sets how much of a section must intersect before it activates. Pass a comma-separated list to fire at multiple steps, e.g. "0.1,0.5,1".

If the page changes after load (sections appended, links rewritten), grab the instance and call refresh().

const el = document.querySelector('[data-bs-spy="scroll"]');
const spy = Stisla.bs.ScrollSpy.getInstance(el);
spy.refresh();
